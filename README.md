## `> what is this`

You write an email. AI helps you draft it. Then it comes out sounding like **a language model wrote it** вҖ” and everyone can tell.

HumanizeMail fixes that. Paste your draft, pick a voice, get back something that sounds like *you* actually typed it at 2pm between meetings.

```diff
- "I hope this email finds you well. I wanted to reach out and explore 
-  potential synergies and delve into ways we could collaborate going forward."

+ "Hope you're doing well вҖ” wanted to see if there's something worth 
+  building together here."
```

## `> provider chain`

No single point of failure. If one provider is rate-limited, it automatically tries the next:

```
Request
   в”Ӯ
   в”ңв”Җв–¶  Groq (llama-3.3-70b)        вҶҗ fastest
   в”Ӯ         в”Ӯ quota hit?
   в”ңв”Җв–¶  OpenRouter (llama-3.3-70b)  вҶҗ fallback
   в”Ӯ         в”Ӯ quota hit?
   в””в”Җв–¶  Gemini (gemini-2.0-flash)   вҶҗ last resort
```

## `> run locally`

```bash
# 1. Clone
git clone https://github.com/trinathone/humanize-mail
cd humanize-mail

# 2. Install
npm install

# 3. Configure вҖ” copy and fill in at least one key
cp .env.example .env.local

# 4. Start API server + frontend
node dev-server.js &   # API on :3000
npm run dev            # Vite on :5173
```

Open **http://localhost:5173**

## `> deploy your own`

```bash
# 1. Fork this repo, then:
npm i -g vercel
vercel --prod

# 2. Set env vars in Vercel dashboard:
#    GROQ_API_KEY, OPENROUTER_API_KEY, or GEMINI_API_KEY
#    APP_PASSWORD  (optional вҖ” locks the app with a password)
```
